事例

Acme Co 株式会社、コンビニ決済と LINE チェックアウトを 8 週間で公開。

200 店舗を展開する日本の専門小売業が、QORTEX 上でコマーススタックを再構築。何が変わり、どう公開され、最初の 90 日で何を達成したのか。

8 週間
キックオフから本番トラフィック開始まで。
+47%
Web 比、LINE 経由のチェックアウト転換率。
背景

200 店舗、3 つのレガシーシステム、不満の残るチェックアウト。

Acme Co 株式会社は、日本国内で 200 店舗の専門小売を運営し、月間 500 万 MAU の EC サイトを展開。QORTEX 導入前のコマーススタックは、12 年前のカスタムプラットフォームをグローバル ERP、ローカル決済ゲートウェイ、別系統の LINE ボットに接続したもの。3 つのシステム、3 つのデータベース、注文が迷子になる場所も 3 箇所。

チェックアウト離脱率は 78%。コンビニ注文は 4 件に 1 件が照合エラー。LINE と Web は連携せず、カスタマーサポートは半日をシステム間の翻訳に費やしていました。

課題

「コンビニと LINE チェックアウトが必要だった。1 年ではなく、四半期で。」

役員会から与えられた期限は 1 つの商戦期。標準的なアプローチ（一からの再プラットフォーム化、1 年間のデュアル稼働、移行）は選択肢にありませんでした。CFO は数字を、COO は日付を、E コマース責任者は次の 3 四半期を統合債務に費やさずに両方を実現することを求めていました。

譲れない要件は 2 つ:

  • ずれないコンビニ照合。 ローソン、ファミリーマート、セブン-イレブンで決済されたすべての注文が、人手を介さず精算される必要があった。
  • LINE、Web、店頭をまたぐ単一の顧客 ID。 LINE で会員登録した顧客が、店頭レジで自分のロイヤルティ残高を確認できる必要があった。
ソリューション

QORTEX の 3 モジュール。標準連携。1 つの設定済みワークフロー。

カスタマーアイデンティティ、受注管理、マーケティングオートメーションを導入 — 譲れない 2 要件を解決する最小構成。残りはフェーズ 2 に。

カスタマーアイデンティティ

LINE、Web、200 店舗の POS をまたぐ統合プロファイル。すべてのフォームで JIS 住所体系検証、漢字 ⇄ ローマ字を標準装備。

受注管理

コンビニ、Pay-easy、代引を標準連携。決済ゲートウェイのエクスポートに対する夜次照合で、手動マッチング不要。

マーケティングオートメーション

LINE メッセージング API が顧客プロファイルと連動。カート放棄や季節キャンペーンを事前テンプレートで提供。

成果

8 週間で稼働、LINE 転換率 +47%、コンビニ照合ずれゼロ。

本番トラフィックは 8 週目に開始。稼働から 4 か月、コンビニ照合は一度も手動マッチングを必要としていません。LINE チェックアウトの転換率は Web 比 +47%。カスタマーサポートの負荷は最初の四半期で 31% 減少。

フェーズ 2 — パーソナライゼーション、PIM、コンテンツ管理 — は Q3 に進行中。

QORTEX のおかげで、コンビニと LINE のチェックアウトを 8 週間で公開できた。
